🚀 Overview & Motivation

While many LLM environments focus on web browsing or puzzle games, there is a critical gap in evaluating an agent's ability to manage dynamic, real-world backend infrastructure. DIME models a mesh-topology graph of compute nodes experiencing realistic distributed systems dynamics.

An LLM agent must act as an automated SRE, observing system telemetry (CPU, queues, latency) and taking decisive management actions (routing, scaling, throttling) to maintain cluster stability and prevent catastrophic cascading failures.

🧮 Simulation Dynamics & Mathematical Models

  • Stochastic Traffic Formulation: Request arrivals are modeled using a variable-rate Gaussian distribution (approximating a Poisson process with burst potential) to simulate unpredictable real-world web traffic.
  • Non-Linear Congestion (Latency): End-to-end latency is calculated using an exponential moving average (EMA). It incorporates base network delay, linear queue delay, and a non-linear (quadratic) CPU pressure penalty (latency ∝ CPU²).
  • Stochastic & Deterministic Failures: Nodes fail deterministically if sustained above 90% CPU for 3 consecutive steps. However, they also face a probabilistic risk of failure when operating under heavy load (>85% CPU), simulating hardware degradation.
  • Cascading Load Redistribution: When a node dies, its load and pending queues are violently redistributed to its adjacent neighbors in the mesh graph, actively simulating the physics of a cascading cluster collapse.

🎯 Graded Tasks

Task 1 — Traffic Spike Recovery (Easy)

System receives 3× normal request rate. Keep latency below 50ms and maintain uptime. Graded on latency control and action efficiency.

Task 2 — Single Node Failure (Medium)

A node fails mid-episode. Agent must restart it and maintain >80% system uptime. Graded heavily on Mean Time To Repair (MTTR).

Task 3 — Cascading Failure Prevention (Hard)

Two connected nodes are near-critical. Agent must proactively reroute traffic before the failure chain triggers. Prevention is rewarded over recovery.

Task 4 — Flash Crowd Meltdown (Expert)

Unprecedented 5× traffic surge. Pure survival scenario requiring aggressive, simultaneous use of scale_up and throttle to prevent total cluster collapse.
